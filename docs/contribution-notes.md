# Contribution Notes

## Purpose

This file explains how updates should be made to the repository in a clean and consistent way.

## Update Guidelines

- Keep each change focused
- Use clear commit messages
- Update related documentation when needed
- Avoid random placeholder content
- Keep folder structure organized
- Review files before committing changes

## Commit Message Examples

Good examples:

- Add setup instructions
- Improve evaluation notes
- Update project checklist
- Add sample results table

Avoid examples:

- update
- fix
- test
- random change
- final final

## Notes

Clear contribution habits make the repository easier to review and maintain over time.
